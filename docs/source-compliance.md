# 공식 출처 크롤링 점검 기록

최종 갱신: 2026-05-26

이 서비스는 공식 소비자정보의 원문을 대체하지 않고, 제목/요약/태그/발행일/원문 URL/수집 시각을 저장해 검색 가능하게 만드는 것을 목표로 한다.

## 공통 운영 원칙

- 운영 크롤링은 하루 1회 이하로 시작한다.
- 상세 원문 전체를 대량 복제하지 않는다.
- 원문 URL, 출처명, 수집 시각을 항상 저장한다.
- 게시판 목록과 상세 페이지 구조가 바뀔 수 있으므로 출처별 파서는 실패해도 전체 서버를 중단하지 않아야 한다.
- 대량 수집, 우회 수집, 로그인/인증 우회, 비공개 API 호출은 하지 않는다.

## 출처별 확인 대상

| 출처 | 크롤링 대상 | robots.txt | 이용 안내/조건 | 현재 정책 |
| --- | --- | --- | --- | --- |
| 소비자24 | 비교공감, 리콜정보 | `https://www.consumer.go.kr/robots.txt` | 소비자24 사이트 이용 안내 확인 필요 | 목록 중심 보수 수집, 원문 링크 저장 |
| 한국소비자원 | 보도자료, 비교정보, 안전주의 | `https://www.kca.go.kr/robots.txt` | 한국소비자원 누리집 이용 안내 확인 필요 | 목록 중심 보수 수집, 원문 링크 저장 |
| 공정거래위원회 | 보도자료, 소비자정책 안내 | `https://www.ftc.go.kr/robots.txt` | `https://www.ftc.go.kr/www/contents.do?key=516` | 목록 중심 보수 수집, 원문 링크 저장 |

## 구현 반영 사항

- `backend/app/crawler.py`는 소비자24, 한국소비자원, 공정거래위원회별 파서 진입점을 분리한다.
- URL 중복은 `documents.url` unique constraint로 방지한다.
- 발행일은 게시글 행 주변 텍스트에서 `YYYY-MM-DD`, `YYYY.MM.DD`, `YYYY년 MM월 DD일` 패턴을 추출해 저장한다.
- 출처 구조가 불안정한 경우에도 제목, 원문 URL, 출처, 수집 시각을 우선 보존한다.

## 운영 전 미해결 체크

- 각 사이트의 `robots.txt`를 운영 배포 직전에 다시 확인한다.
- 각 사이트의 이용 조건에서 자동 수집 제한, 저작권, 출처 표기 조건을 확인한다.
- 실제 운영 로그에서 요청 빈도와 실패율을 확인한 뒤 크롤링 주기를 조정한다.
