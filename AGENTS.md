# AGENTS.md

## 프로젝트

이 프로젝트는 SNS에서 본 “자취생이 괜찮은 물건을 고르는 방법”이라는 아이디어에서 출발한 서비스다.

작업명은 **혼자살림 레이더**다.

핵심은 국가와 공공기관이 발행한 소비자 비교정보, 구매 가이드, 안전 정보 등을 크롤링해서 저장하고, 사용자가 나중에 검색할 수 있는 공식 소비자정보 아카이브를 만드는 것이다.

서비스 흐름은 학교 공지사항 크롤러와 유사하다.

1. 공식 소비자정보 출처를 크롤링한다.
2. 문서 정보를 정규화한다.
3. 데이터베이스에 저장한다.
4. 프론트엔드에서 저장된 아카이브를 검색하게 한다.

대상 사용자는 주로 한국의 1인 가구와 자취생이다. “무엇을 사야 하지?”라는 막연한 질문에 광고성 추천이 아니라 공공기관 비교정보를 근거로 찾아볼 수 있게 하는 것이 목표다.

## 제품 원칙

- 광고, 제휴 링크, 쇼핑몰 랭킹을 중심에 두지 않는다.
- 출처가 불분명한 생활 팁을 원문처럼 저장하지 않는다.
- 공식 출처 URL과 수집 시각을 남겨 사용자가 원문을 확인할 수 있게 한다.
- 긴 원문을 그대로 복제하기보다 제목, 요약, 태그, 발행일, 원문 링크를 구조화해서 저장한다.
- “자취생 추천템” 페이지가 아니라 “공식 비교정보 검색 archive”로 설계한다.

## 기술 스택

- 프론트엔드: React + Vite
- 백엔드: FastAPI
- 로컬 DB: SQLite
- 운영 DB: Neon Postgres
- 스케줄러: GitHub Actions
- 백엔드 호스팅: Render
- 프론트엔드 호스팅: Vercel

## 디자인 시스템

디자인 토큰은 필수다. 공통 값은 `src/design-tokens.css`에 두고 `src/styles.css`에서 참조한다.

토큰 그룹:

- color
- typography
- spacing
- radius
- border
- shadow
- motion

UI는 카드뉴스처럼 보이지 않게 만든다. 검색, 필터, 목록, 문서 상세, 출처 확인이 잘 되는 조용하고 실용적인 정보 서비스 톤을 우선한다.

## 백엔드 규칙

FastAPI 백엔드가 크롤링, 저장, 검색을 책임진다.

중요 엔드포인트:

- `GET /health`
- `GET /api/sources`
- `GET /api/documents`
- `POST /api/admin/seed`
- `POST /api/admin/crawl`

크롤러가 저장하는 문서는 다음 정보를 보존해야 한다.

- 출처 이름
- 원문 URL
- 문서 유형
- 제목
- 요약
- 태그
- 발행일, 확인 가능한 경우
- 수집 시각

중복 저장을 막기 위해 원문 URL의 고유성을 기준으로 삼는다.

## 출처 정책

공식 소비자정보는 시간이 지나며 바뀔 수 있다. 대량 원문 복제보다 canonical URL, 발행일, 수집 시각을 정확히 저장하는 쪽을 우선한다.

우선 출처:

- 소비자24
- 공정거래위원회
- 한국소비자원

운영 크롤링 전에 각 사이트의 `robots.txt`와 이용 조건을 확인한다. 크롤링 빈도는 보수적으로 유지한다.

## 검증

인계 전에는 다음을 확인한다.

- `npm run build` 실행
- 백엔드 코드 파싱 또는 실행 확인
- `/health` 확인
- 로컬 데이터 seed 실행
- `/api/documents` 확인
- README와 배포 안내가 실제 아키텍처와 일치하는지 확인
